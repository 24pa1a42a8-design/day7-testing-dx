# Revision Answers

## 1. Why are tests important?
They ensure system reliability and prevent production issues.

## 2. Problems without testing?
Data corruption, automation failure, incorrect analytics.

## 3. Why asynchronous processing?
Improves performance and user experience.

## 4. Synchronous vs Asynchronous?
Synchronous runs immediately.
Asynchronous runs in background.

## 5. Why version control?
Tracks code changes and supports collaboration.

## 6. Why GitHub important?
Code sharing, collaboration, backups.

## 7. Why DX useful?
Modern Salesforce development and team workflow support.

## 8. How Flows, Triggers, Validation Rules work together?
Validation checks data, flows automate processes, triggers execute backend logic.

## 9. Why test business logic carefully?
Incorrect logic affects operations and analytics.

## 10. Why developer workflow important?
Ensures scalable and maintainable enterprise development.
